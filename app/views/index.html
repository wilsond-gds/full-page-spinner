Full page indeterminate waiting prototypes

These are intial explorations informed by:

  • Loading spinner discussion
  • Indeterminate progress indicator discussion
  • Meta refresh is to be avoided for accessibility reasons
  • Our users would benefit from a spinner having a minimum runtime

  • Meta refresh version

    This version updates the page content to reassure users the system is working. However, using a meta refresh removes control from the user, so is not ideal for accessibility.

  • User controlled version

    This version updates the page content to show a continue button, and also sketches out a error state if the operation doesn’t complete. However it doesn’t consider how the user could interact with the system if JavaScript is disabled.
